# 4. Ports Must Be Configurable

Yes — this is currently a gap.

Your current backend launch profile still has fixed local URLs like:

```json
"applicationUrl": "http://localhost:5063"
```

And your frontend API client currently has a fallback like:

```ts
import.meta.env.VITE_API_BASE_URL || "http://localhost:5063"
```

The frontend part is already partly good because it supports `VITE_API_BASE_URL`, but the fallback is still static. The backend CORS also currently allows a static frontend origin, `http://localhost:5173`, which must become configurable before publishing.

You need environment-based configuration:

## Backend

Use:

```bash
ASPNETCORE_URLS=http://+:8080
PLANTPROCESS_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
ConnectionStrings__PlantProcessDb=Host=localhost;Port=5432;Database=plantprocessiq;Username=plantprocess;Password=plantprocess123
```

Backend `appsettings.json` should include:

```json
{
  "PlantProcess": {
    "AllowedOrigins": [
      "http://localhost:5173"
    ]
  }
}
```

And CORS should read from config, not hardcoded string.

## Frontend

Create `.env.development`:

```env
VITE_API_BASE_URL=http://localhost:5063
```

Create `.env.production`:

```env
VITE_API_BASE_URL=https://your-api-domain.com
```

Run FE with variable port:

```bash
npm run dev -- --host 0.0.0.0 --port 5173
```

Or define:

```json
"scripts": {
  "dev": "vite --host 0.0.0.0 --port ${VITE_PORT:-5173}"
}
```

On Windows, better use `cross-env` or use `.env`.

## Docker Compose

Use variables:

```yaml
services:
  api:
    ports:
      - "${API_PORT:-5063}:8080"
    environment:
      ASPNETCORE_URLS: "http://+:8080"
      ConnectionStrings__PlantProcessDb: "Host=db;Port=5432;Database=${POSTGRES_DB};Username=${POSTGRES_USER};Password=${POSTGRES_PASSWORD}"
      PlantProcess__AllowedOrigins__0: "http://localhost:${WEB_PORT:-5173}"

  web:
    ports:
      - "${WEB_PORT:-5173}:5173"
    environment:
      VITE_API_BASE_URL: "http://localhost:${API_PORT:-5063}"

  db:
    ports:
      - "${DB_PORT:-5432}:5432"
```

Then `.env`:

```env
API_PORT=5063
WEB_PORT=5173
DB_PORT=5432
POSTGRES_DB=plantprocessiq
POSTGRES_USER=plantprocess
POSTGRES_PASSWORD=plantprocess123
```

This gives you full flexibility when publishing.

# Very Important Design Rule

Never query Oracle/MSSQL/Excel directly from the dashboard.

Bad design:

```text
React dashboard → Backend → Oracle/MSSQL/Excel directly every time
```

Correct design:

```text
Source systems → Import/Staging → Canonical model → Analytics marts → Dashboard
```

Because if the dashboard queries live source systems directly:

```text
1. Dashboard will be slow.
2. Customer source systems may be overloaded.
3. Excel/API sources are unreliable for live dashboarding.
4. Correlation/risk logic becomes inconsistent.
5. You lose audit/replay ability.
```
