Engineer Brief

PlantProcess IQ

Read-only manufacturing intelligence layer for process-to-quality investigation.

What it is

PlantProcess IQ connects plant data, creates a safe staging copy, maps customer-specific schemas into a generic canonical manufacturing model, and supports dashboards, data-quality checks, risk scoring, correlation, investigation, ML readiness, and Data Diagnostic reporting.

What it is not

  • Not MES replacement.
  • Not SCADA replacement.
  • Not Level 2 / PLC control.
  • Not BI-only dashboarding.
  • Not steel-only.

Core lifecycle

  • Connect source data.
  • Stage a read-only data copy.
  • Map schema into canonical entities.
  • Monitor import and analysis jobs.
  • Analyze quality, risk, correlation and genealogy.
  • Report findings and ML readiness.

Current intelligence

Current intelligence is based on rule-based risk scoring, data-quality scanning, statistical correlation, suspected contributor ranking, and investigation workflows.

ML honesty

No trained production ML model is active. ML training starts only after validated labeled historical data, genealogy coverage, feature vectors, and readiness gates are sufficient.

Connector honesty

Only tested connectors should be marked available. CSV is safe for starter demos. PostgreSQL is available only when environment proof is green. SQL Server and Oracle remain planned until implementation and tests pass.
